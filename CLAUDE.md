## README Architecture

The README uses modern GitHub profile techniques for maximum visual impact and SEO:

1. **Animated typing header** — `readme-typing-svg` with dark/light mode via `<picture>` element
2. **Contact badges** — shields.io badges (Portfolio, LinkedIn, Email) + profile view counter
3. **About me** — TypeScript code block as creative intro + narrative summary
4. **Tech stack icons** — Visual grid via `skillicons.dev` with collapsible detailed table
5. **Production projects** — Rich HTML table cards with per-project tech badges and store links

7. **GitHub stats** — Stats card + streak stats + top languages, all with dark/light mode support
8. **Snake animation** — Contribution graph snake (generated by GitHub Action every 6 hours)
9. **CTA footer** — Availability + contact button
10. **SEO keywords** — Hidden HTML comment block with ATS/recruiter search terms

## Key Conventions

### Language
- The README is written in **Spanish**. Maintain Spanish for all user-facing content unless explicitly told otherwise.

### Formatting
- **Dark/light mode support** using `<picture>` + `<source media="(prefers-color-scheme: ...)">` for all dynamic images.
- **shields.io badges** for contact links and per-project technology tags.
- **skillicons.dev** for the main tech stack visual grid.
- **HTML tables** for project cards (richer than Markdown tables).
- **`<details>` / `<summary>`** for collapsible sections.
- Sections separated by horizontal rules (`---`).

### Dynamic Elements (Third-party Services)
- `readme-typing-svg.demolab.com` — Animated typing header
- `skillicons.dev` — Technology icon grid
- `github-readme-stats.vercel.app` — GitHub stats & top languages cards
- `streak-stats.demolab.com` — Contribution streak stats
- `komarev.com/ghpvc` — Profile view counter
- `img.shields.io` — All badge images

### GitHub Actions
- **Snake workflow** (`.github/workflows/snake.yml`): Runs every 6 hours via `Platane/snk`, generates snake SVGs to the `output` branch. Must be manually triggered the first time via `workflow_dispatch`.

## Working with This Repository

### Making Changes
Typical edits involve:
- Adding or updating production projects (update the HTML table cards)
- Updating the tech stack (update both `skillicons.dev` URL and the detailed table)
- Refining professional messaging or adding new sections
- Updating contact/social links

### Git Workflow
- **Default branch:** `main`
- Commit messages should be clear and descriptive.
- No build, test, or lint steps required before committing.
- The `output` branch is auto-generated by GitHub Actions (snake SVGs) — do not edit manually.

### What NOT to Do
- Do not change the language from Spanish without explicit instruction.
- Do not remove or alter external project links without confirmation.
- Do not manually edit the `output` branch (auto-generated).
- Do not break `<picture>` elements — they provide dark/light mode support.
